# Frontend Design Elite

## Objetivo
Elevar cualquier entrega frontend al estandar de estudio premium: identidad visual diferenciada, UX clara, implementacion limpia y comportamiento solido en desktop y mobile.

## Flujo de trabajo
1. Definir contexto minimo antes de disenar: objetivo del negocio, audiencia principal, accion esperada y restricciones tecnicas.
2. Elegir una direccion de arte unica (no neutra) y justificarla en 2-3 frases. Consultar `references/art-directions.md` si hace falta.
3. Construir un mini sistema visual antes de maquetar: tipografia, paleta, escala de espaciado, radios, sombras y reglas de estado.
4. Disenar composicion con ritmo: jerarquia evidente, puntos focales claros, contraste de bloques y fondos con profundidad.
5. Aplicar motion con intencion: 2-4 animaciones utiles (entrada, scroll reveal, cambio de estado, feedback de accion).
6. Implementar con calidad de produccion: responsive real, accesibilidad minima y estilos mantenibles.
7. Autoevaluar con `references/elite-rubric.md` y corregir hasta cumplir el umbral antes de entregar.

## Reglas obligatorias
- Evitar apariencia de plantilla: no usar layout o paleta generica sin una direccion artistica definida.
- Evitar tipografia por defecto (`Inter`, `Roboto`, `Arial`, `system-ui`) salvo que el proyecto existente ya lo imponga.
- Evitar fondo plano unico; usar capas visuales (gradientes, texturas suaves, shapes, luces o patrones).
- Mantener coherencia entre tono visual y objetivo de producto (ejemplo: fintech sobrio, lifestyle expresivo).
- Preservar patrones si se trabaja sobre un sistema de diseno existente; innovar dentro de sus limites.

## Contrato de salida
Incluir siempre en la respuesta final:
1. Concepto visual elegido y por que encaja con el objetivo.
2. Tokens clave (colores, tipografias, espaciado, radios, sombras, duraciones).
3. Implementacion completa (HTML/CSS o React + estilos) lista para ejecutar.
4. Checklist breve de calidad: responsive, accesibilidad, consistencia visual y deuda tecnica pendiente.

## Checklist tecnico minimo
- Confirmar viewport mobile (>=360px) y desktop (>=1280px) sin roturas.
- Asegurar contraste legible en textos criticos y estados de foco visibles en elementos interactivos.
- Limitar complejidad visual innecesaria si afecta rendimiento percibido.
- Entregar codigo legible, con variables CSS/tokens y nombres semanticos.

## Referencias
- Consultar `references/art-directions.md` para elegir direccion visual concreta segun producto y audiencia.
- Consultar `references/elite-rubric.md` para puntuar el resultado y detectar mejoras antes de cerrar.
